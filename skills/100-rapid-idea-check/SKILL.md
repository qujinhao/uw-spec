## 五维分析框架

### 1. 需求真实性（七宗罪人性逻辑）

| 维度 | 问题 | 强相关标志 |
|------|------|-----------|
| 傲慢 | 是否满足优越感/身份认同？ | 社交展示、等级体系 |
| 懒惰 | 是否省时省力？ | 一键操作、自动化 |
| 贪婪 | 是否让用户觉得"赚到了"？ | 优惠、返利、免费 |
| 暴怒 | 是否解决麻烦/负面情绪？ | 投诉、吐槽、解压 |
| 嫉妒 | 是否解决怕落后的焦虑？ | 排行榜、限时 |
| 暴食 | 是否创造稀缺感/复购？ | 限量、会员 |
| 色欲 | 是否有视觉/感官吸引力？ | 颜值、体验 |

**判断**：≥3个维度强相关 → 真实需求

### 2. 用户价值（俞军公式）

```
用户价值 = 新体验 - 旧体验 - 替换成本
```

| 评估项 | 标准 |
|--------|------|
| 新体验 >> 旧体验 + 替换成本 | 值得做 |
| 新体验 ≈ 旧体验 | 不值得 |
| 替换成本过高 | 需降低门槛 |

### 3. 竞争格局

| 情况 | 判断 |
|------|------|
| 已有巨头且做得很好 | 避开 |
| 有玩家但体验差 | 有机会 |
| 完全空白 | 可能蓝海/没市场 |

### 4. 商业化可行性

| 付费模式 | 可行性 |
|----------|--------|
| 清晰的付费方和动机 | 可行 |
| 依赖广告 | 需验证 |
| 免费习惯根深蒂固 | 困难 |

### 5. 独立开发者适配度

| 评估项 | 适合 | 不适合 |
|--------|------|--------|
| 开发周期 | 1-2个月MVP | 需团队/大资金 |
| 技术难度 | 单人可搞定 | 需专业团队 |
| 冷启动 | 有明确获客渠道 | 依赖大量投放 |

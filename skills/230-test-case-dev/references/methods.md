# 测试设计方法

## 等价类划分法

将输入数据划分为若干等价类，从每个等价类中选取代表性数据作为测试用例。

### 示例：年龄输入框
- 有效等价类：1-150
- 无效等价类：<1, >150, 非数字

| 用例 | 输入 | 预期结果 |
|------|------|----------|
| 有效值 | 25 | 验证通过 |
| 边界最小 | 1 | 验证通过 |
| 边界最大 | 150 | 验证通过 |
| 无效小 | 0 | 提示错误 |
| 无效大 | 151 | 提示错误 |
| 非数字 | abc | 提示错误 |

## 边界值分析法

针对输入范围的边界值进行测试，因为错误往往发生在边界。

### 边界值选取
最小值 → 略高于最小值 → 正常值 → 略低于最大值 → 最大值

### 示例：用户名长度（6-20字符）
| 用例 | 输入 | 预期结果 |
|------|------|----------|
| 低于最小 | 5字符 | 提示错误 |
| 最小边界 | 6字符 | 验证通过 |
| 正常值 | 10字符 | 验证通过 |
| 最大边界 | 20字符 | 验证通过 |
| 超过最大 | 21字符 | 提示错误 |

## 场景法（流程分析法）

基于用户业务流程设计测试用例，覆盖基本流和备选流。

### 示例：电商下单流程
```
基本流：浏览商品 -> 加入购物车 -> 结算 -> 支付 -> 订单完成

备选流1：商品库存不足
备选流2：优惠券使用失败
备选流3：支付超时
备选流4：支付失败
```

| 场景 | 流程 | 预期结果 |
|------|------|----------|
| 正常下单 | 基本流 | 订单成功 |
| 库存不足 | 基本流+备选流1 | 提示库存不足 |
| 支付失败 | 基本流+备选流4 | 订单待支付 |

## 错误推测法

基于经验和直觉，推测系统可能存在的缺陷，针对性设计用例。

| 场景 | 测试点 |
|------|--------|
| 空值处理 | 输入为空、null、undefined |
| 特殊字符 | 输入< > ' " &等特殊字符 |
| 超长输入 | 输入超过最大长度 |
| 并发操作 | 多用户同时操作同一数据 |
| 网络异常 | 断网、超时、弱网环境 |
| 数据边界 | 最大值、最小值、零值 |

## 安全测试设计方法

### 威胁建模（STRIDE）

| 威胁类型 | 说明 | 测试方向 |
|---------|------|---------|
| Spoofing（欺骗） | 冒充其他用户身份 | 认证绕过、Token伪造 |
| Tampering（篡改） | 修改数据或通信内容 | 参数篡改、中间人攻击 |
| Repudiation（否认） | 否认执行过操作 | 审计日志完整性 |
| Info Disclosure（信息泄露） | 敏感信息暴露 | 错误信息泄露、日志泄露 |
| DoS（拒绝服务） | 使系统不可用 | 大量请求、资源耗尽 |
| Elevation（提权） | 获取未授权权限 | 水平越权、垂直越权 |

### OWASP Top 10 映射

| OWASP类别 | 测试用例方向 | 适用接口 |
|-----------|------------|---------|
| A01 权限控制失效 | 水平越权（访问他人数据）、垂直越权（低角色访问高角色功能） | 所有业务接口 |
| A02 加密失败 | 敏感数据明文传输、弱密码哈希 | 登录、个人信息接口 |
| A03 注入 | SQL注入、XSS、命令注入 | 所有含输入参数的接口 |
| A04 不安全设计 | 业务逻辑漏洞、竞态条件 | 订单、支付、库存接口 |
| A05 安全配置错误 | 默认密码、调试接口暴露、CORS配置 | 全局配置接口 |
| A06 过期组件 | 已知漏洞依赖 | 依赖扫描（Trivy） |
| A07 认证失败 | 暴力破解、弱密码策略、Session固定 | 登录、注册、密码重置 |
| A08 数据完整性失败 | 反序列化漏洞、不安全的CI/CD | 文件上传、导入接口 |
| A09 日志监控不足 | 敏感操作无日志 | 关键业务操作 |
| A10 服务端请求伪造 | SSRF攻击内部服务 | URL输入、回调接口 |

### 安全测试用例设计步骤

1. **识别攻击面**：列出所有接口及其参数
2. **映射OWASP类别**：每个接口对应的安全风险类别
3. **设计攻击向量**：针对每个风险点构造攻击载荷
4. **定义预期安全行为**：请求被拦截/数据被转义/返回403
5. **ZAP扫描覆盖**：自动化扫描 + 手动验证关键场景

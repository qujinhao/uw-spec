# 功能文档更新

从当前目录向上查找 `project-info.md`，最多 3 层，找到后记为 `PROJECT_ROOT`。详见 [检测方法与前置检查](../0-init/references/project-env-check.md)。**未找到** → 提示用户先执行 `0-init`。

## 执行流程

### 1. 运维文档更新

**更新内容**:

| 文档 | 更新点 |
|------|--------|
| 部署手册 | 新增配置项、部署步骤 |
| 监控手册 | 新增监控指标、告警规则 |
| 故障处理 | 新增故障场景处理 |
| 配置管理 | 新增配置参数说明 |

**生成更新记录**:

**文件位置**: `manual/ops-manual/`

**内容**:
```markdown

- 更新日期: {YYYY-MM-DD}
- 功能名称: {功能名称}

- {配置项}: {说明}

## 监控手册更新
### 新增指标
- {指标名}: {说明}

### 告警规则
- {规则名}: {条件}

- [x] 已合并到 deployment.md
```

## ⚠️ 完成验证（强制，全自动执行）

开发工作完成后，**立即按以下顺序自动执行**：

1. **强制调用** `651-feature-doc-review`
2. 如果评审不通过（< 95），自动修复问题，然后回到步骤 1（最多 5 轮）
3. 直到评审通过（≥ 95），**才向用户报告最终结果**

> **此流程全自动执行：中间不暂停、不询问、不汇报。**
> **未收到通过确认前，禁止结束本技能任务。**

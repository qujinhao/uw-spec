# 功能测试验收检查清单

## 测试脚本评审

| 检查项 | 通过标准 |
|--------|---------|
| 方案一致性 | 测试用例覆盖技术方案中所有测试点 |
| 用例完整性 | 正常/异常/边界场景覆盖，≥90%需求点 |
| 测试数据 | Mock数据合理，无硬编码敏感信息 |
| 代码质量 | 命名规范，断言明确，无冗余 |
| 自动化质量 | 测试独立可重复，无执行顺序依赖 |

## 验收测试检查

| 检查项 | 通过标准 |
|--------|---------|
| API测试 | 本功能相关API 100%通过 |
| E2E测试 | 本功能相关流程 100%通过 |
| 核心流程回归 | 主业务流程+关联功能+权限流程 100%通过 |
| 测试通过率 | ≥95% |

## 安全扫描检查

| 检查项 | 通过标准 |
|--------|---------|
| 安全漏洞 | Critical=0 |
| 依赖漏洞 | Critical=0 |
| 敏感信息 | 无密钥密码泄露 |

## 性能扫描检查

| 检查项 | 通过标准 |
|--------|---------|
| SQL性能 | 无慢查询，索引使用正确 |
| API性能 | 响应时间达标 |
| 前端性能 | 资源加载、渲染性能达标 |

## 文档更新检查

| 检查项 | 通过标准 |
|--------|---------|
| 运维文档 | 部署、监控、故障处理已更新 |
| 用户手册 | 操作说明、FAQ已更新 |
| 需求文档 | 功能清单、变更记录已更新 |
| 主文档 | 所有变更已合并 |

## 通过标准

| 等级 | 评分 | 条件 |
|------|------|------|
| 通过 | ≥95分 | 测试通过率≥95%，Critical漏洞0个，核心流程100%通过，人工确认通过 |
| 不通过 | <95分 | 测试通过率<95%或存在Critical漏洞或核心流程失败 |

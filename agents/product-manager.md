# 产品经理

## 角色定位

负责将用户需求转化为产品功能，确保产品方向正确、用户体验优秀。

## 核心职责

| 职责 | 说明 |
|------|------|
| 需求分析 | 深入理解用户痛点，收集和整理用户需求，定义功能优先级 |
| 产品设计 | 设计产品功能流程、规划产品路线图、制定迭代计划 |
| 验收标准 | 为每个功能定义明确的验收标准（AC），确保可测试性 |
| 最终确认 | `630-feature-test`/`730-bugfix-test` 阶段的最终验收确认和上线决策 |

> 需求规划流程详见 [110-requirement-planning](../skills/110-requirement-planning/SKILL.md) 和 [610-feature-clarify](../skills/610-feature-clarify/SKILL.md)。

## 协作关系

| 协作对象 | 协作内容 |
|---------|---------|
| 系统架构师 | 确认技术可行性、澄清需求细节 |
| 项目经理 | 制定迭代计划 |
| 测试工程师 | 定义验收标准 |

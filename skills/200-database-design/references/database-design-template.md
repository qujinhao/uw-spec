## 4. 实体详细设计

### 4.1 guest_user（用户表）

**实体说明**: C端用户信息

**数据规模**:
| 指标 | 数值 |
|------|------|
| 初始数据量 | 10万 |
| 年增长量 | 50万 |
| 3年后数据量 | 160万 |

**DDL定义**: 参见 database-ddl.sql → guest_user

**非功能设计**:
- 分表策略: 单表，数据量<500万不分表
- 缓存策略: 热点用户数据缓存到Redis，TTL=1小时
- 归档策略: 无

---

### 4.2 order_info（订单表）

**实体说明**: 订单主表

**数据规模**:
| 指标 | 数值 |
|------|------|
| 初始数据量 | 100万 |
| 年增长量 | 1000万 |
| 3年后数据量 | 3100万 |

**DDL定义**: 参见 database-ddl.sql → order_info

**非功能设计**:
- 分表策略: 按月份分表，order_info_YYYYMM
- 缓存策略: 最近7天订单缓存到Redis
- 归档策略: 3个月前已完成订单归档到 order_info_archive

---

### 4.3 order_item（订单明细表）

**实体说明**: 订单商品明细

**数据规模**:
| 指标 | 数值 |
|------|------|
| 初始数据量 | 300万 |
| 年增长量 | 3000万 |
| 3年后数据量 | 9300万 |

**DDL定义**: 参见 database-ddl.sql → order_item

**非功能设计**:
- 分表策略: 与 order_info 同维度分表
- 缓存策略: 无
- 归档策略: 与订单主表同步归档

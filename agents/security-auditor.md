# 安全审计员

## 角色定位

负责全面审计系统的安全性，包括代码安全、基础设施安全、数据安全和合规性，识别安全风险并提出修复建议。

## 核心职责

| 职责 | 说明 |
|------|------|
| 代码安全审计 | 审计源代码安全漏洞（OWASP Top 10）、审查第三方依赖安全 |
| 基础设施安全 | 审计云资源配置、网络安全设置、容器和 K8s 安全 |
| 数据安全审计 | 检查数据加密措施、数据访问控制、数据备份策略 |
| 合规性审计 | 验证身份认证机制、权限控制实现、日志和监控合规 |

> 安全测试检查清单详见 [230-test-case-dev](../skills/230-test-case-dev/references/security-checklist.md)。

## 协作关系

| 协作对象 | 协作内容 |
|---------|---------|
| 系统架构师 | 确认安全架构，仲裁争议 |
| Java负责人 | Java 代码安全问题 |
| JS前端负责人 | JavaScript 代码安全问题 |
| 运维工程师 | 基础设施安全配置 |
| 项目经理 | 安全风险评估和排期 |

# 功能开发评审检查清单

## Phase 1: 需求评审

| 检查项 | 通过标准 |
|--------|---------|
| 需求完整性 | 所有验收标准可量化、可测试 |
| 边界情况 | 已识别并记录边界条件 |
| 安全性 | 已识别安全风险点 |
| 影响范围 | 已明确影响的功能模块 |

## Phase 2: 技术方案评审

| 检查项 | 通过标准 |
|--------|---------|
| 数据库设计 | 表结构合理、命名规范、索引充分、DDL可执行、与现有设计兼容 |
| 后端接口设计 | 接口规范、权限注解、DTO设计、Helper签名、与现有代码一致 |
| 前端页面设计 | 页面结构、路由配置、组件设计、字段一致性、与现有页面风格统一 |
| 测试策略 | 测试类型覆盖、场景完整、数据准备方案 |
| 兼容性 | 不破坏现有功能、DDL向后兼容、接口向后兼容 |
| PRD覆盖度 | 技术方案覆盖所有功能需求点 |

## Phase 4: 代码评审

### 后端代码

| 检查项 | 通过标准 |
|--------|---------|
| 方案一致性 | 代码实现与技术方案完全一致，接口/DTO/Helper无偏差 |
| TDD实践 | 覆盖率行≥80%/分支≥70%，无fail()残留，BaseIntegrationTest+@SpringBootTest |
| UniWeb规范 | DaoManager/ResponseData/AuthQueryParam/FusionCache使用正确，禁用Lombok |
| API陷阱 | 16条已验证的API陷阱全部检查（详见210 dev-standards.md） |
| 代码质量 | 单一职责、分层清晰、命名规范、圈复杂度≤10 |
| 安全性 | @Valid校验、SQL参数化、@MscPermDeclare、SaaS租户隔离（saas_id）、modifyDate必设 |
| 影响范围 | 未修改无关模块，变更范围可控 |

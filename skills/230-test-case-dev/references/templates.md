## 2. E2E 界面测试用例模板

### 2.1 单终端用例

```markdown
### TC-E2E-{module}-{seq}: {用例标题}
**终端**: {前端项目名，如 nova-guest-uni}
**角色**: {role}
**优先级**: P0/P1/P2/P3

#### 前置条件
- 登录角色: {role}
- 初始数据: {依赖数据}

#### 测试步骤
| 步骤 | 操作 | 元素定位 | 预期结果 |
|------|------|---------|---------|
| 1 | 导航到xxx | 导航栏/Tab栏中的"xxx"链接 | 页面加载完成 |
| 2 | 点击xxx | `getByRole('button', { name: 'xxx' })` | 弹出对话框 |
| 3 | 填写xxx | `locator('[name="fieldName"]')` 或 `getByPlaceholder('xxx')` | 输入成功 |
| 4 | 点击提交 | `getByRole('button', { name: '提交' })` | 提示成功 |

#### 测试数据
| 字段 | 值 | 说明 |
|------|-----|------|
| xxx | xxx | 正常值 |
```

### 2.2 跨终端用例

```markdown
### TC-E2E-CROSS-{seq}: {跨终端业务流程标题}
**涉及终端**: guest-uni, admin-web, mch-web
**业务场景**: {如：用户下单→管理端审核→商家发货→用户确认收货}
**优先级**: P0

#### 流程步骤
| 步骤 | 终端 | 角色 | 操作 | 元素定位 | 预期结果 |
|------|------|------|------|---------|---------|
| 1 | guest-uni | guest | 提交订单 | `getByRole('button', { name: '提交订单' })` | 订单创建成功，状态=待审核 |
| 2 | admin-web | admin | 审核通过 | `getByRole('button', { name: '审核' })` | 审核通过，状态=待发货 |
| 3 | mch-web | mch | 确认发货 | `getByRole('button', { name: '发货' })` | 发货成功，状态=运输中 |
| 4 | guest-uni | guest | 确认收货 | `getByRole('button', { name: '确认收货' })` | 订单完成，状态=已完成 |

#### 断言点
| 步骤 | 断言方式 | 预期值 |
|------|---------|--------|
| 1 | API响应断言 | order.status = PENDING_REVIEW |
| 2 | 页面元素断言 | `getByText('已审核')` |
| 3 | 页面元素断言 | `getByText('运输中')` |
| 4 | 页面元素断言 | `getByText('已完成')` |
```

## 优先级定义

| 优先级 | 说明 | 覆盖要求 |
|--------|------|----------|
| P0 | 冒烟测试，核心流程 | 100%覆盖 |
| P1 | 核心功能，影响用户体验 | 90%以上覆盖 |
| P2 | 扩展场景，有替代方案 | 70%以上覆盖 |
| P3 | 边界异常，低频场景 | 按需覆盖 |

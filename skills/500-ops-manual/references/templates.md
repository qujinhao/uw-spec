## 2. 部署手册

```markdown
# 部署手册

## 2.1 环境要求

| 服务 | CPU | 内存 | 磁盘 | 数量 |
|------|-----|------|------|------|
| Gateway | 2核 | 4G | 50G | 2 |
| App Service | 4核 | 8G | 100G | 3 |
| MySQL | 8核 | 16G | 500G | 3 |
| Redis | 4核 | 8G | 50G | 6 |

## 2.2 部署流程

```bash
# 1. 创建应用目录
mkdir -p /opt/apps/{gateway,user-service,order-service}
mkdir -p /opt/data/{mysql,redis}
mkdir -p /opt/logs/{gateway,user-service,order-service}

# 2. 部署MySQL主库
docker run -d \
  --name mysql-master \
  -p 3306:3306 \
  -v /opt/data/mysql/master:/var/lib/mysql \
  -e MYSQL_ROOT_PASSWORD=xxxxxx \
  mysql:8.4

# 3. 部署应用服务
docker pull registry.example.com/user-service:1.0.0
docker run -d \
  --name user-service \
  -p 8081:8081 \
  -v /opt/logs/user-service:/app/logs \
  -e SPRING_PROFILES_ACTIVE=prod \
  registry.example.com/user-service:1.0.0
```

## 2.3 健康检查

```bash
curl http://localhost:8081/actuator/health
```
```

## 4. 故障处理手册

```markdown
# 故障处理手册

## 4.1 故障分级

| 级别 | 定义 | 响应时间 | 处理时限 |
|------|------|----------|----------|
| P0 | 系统完全不可用 | 立即 | 30分钟 |
| P1 | 核心功能不可用 | 5分钟 | 1小时 |
| P2 | 部分功能异常 | 15分钟 | 4小时 |
| P3 | 性能下降 | 30分钟 | 8小时 |

## 4.2 常见故障处理

### 服务无响应
**排查步骤**：
1. 检查服务状态：`docker ps | grep service-name`
2. 查看服务日志：`docker logs -f service-name`
3. 检查资源使用：`docker stats service-name`

**处理措施**：
- 服务宕机：重启服务
- OOM：增加内存配置
- 依赖故障：修复依赖服务
```

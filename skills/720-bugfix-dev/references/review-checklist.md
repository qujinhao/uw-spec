# Bug修复开发评审检查清单

## Phase 1: 修复评审

| 检查项 | 通过标准 |
|--------|---------|
| 根因理解 | 根因分析正确，修复针对根因，无潜在类似问题 |
| 复现步骤 | Bug复现步骤清晰、可重复 |
| 影响范围 | 已识别所有受影响的模块 |

## Phase 2: 修复方案评审

| 检查项 | 通过标准 |
|--------|---------|
| 根因理解 | 修复方案针对根因，非表面修补 |
| 修复正确性 | 修复逻辑正确，边界条件处理完整 |
| 彻底性 | 同类问题已一并修复，无遗漏 |
| 数据库变更 | DDL语法正确、向后兼容、有回滚方案 |
| 接口变更 | 不破坏现有接口，兼容性保证 |
| 影响范围 | 变更范围最小化，无不必要修改 |
| 回滚方案 | 回滚步骤明确，数据可恢复 |
